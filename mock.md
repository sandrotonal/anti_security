Görev: Projedeki başarısız Shopier ödeme altyapısını tamamen sistemden kaldırıp, yerine Paddle v2 (Sandbox) entegrasyonunu temiz, güvenli ve TypeScript kurallarına uygun şekilde kurman gerekiyor.Lütfen aşağıdaki adımları sırasıyla ve dosyaları bozmadan uygula:1. Shopier Kalıntılarının Temizlenmesi:api/create-checkout.ts, api/shopier-callback.ts ve api/shopier-return.ts gibi Shopier ile ilgili tüm serverless API backend dosyalarını tespit et ve güvenli bir şekilde sil.  Proje genelinde (özellikle Pricing bileşeninde veya ana sayfalarda) Shopier'e ait ortam değişkenlerini (SHOPIER_API_KEY, SHOPIER_API_SECRET, SHOPIER_WEBHOOK_TOKEN) kullanan tüm fonksiyonları, buton tetikleyicilerini ve referansları kaldır.2. Frontend'e Paddle v2 Entegrasyonu (src/components/SecurifyPricing.tsx):Pricing bileşeninde bulunan "Premium" satın alma butonunu Paddle v2 Overlay Checkout yapısına uyarla.window.Paddle nesnesi için TypeScript declare global arayüzünü ekle.Satın alma butonuna tıklandığında window.Paddle.Environment.set('sandbox'); ve window.Paddle.Initialize({ token: process.env.NEXT_PUBLIC_PADDLE_CLIENT_TOKEN }); adımlarıyla Paddle'ı başlat.window.Paddle.Checkout.open fonksiyonunu çağırarak ödeme penceresini aç. priceId alanına dinamik bir değişken veya string yer tutucu koy.Çok Kritik: customData: { userId: "CURRENT_USER_ID" } yapısını checkout'a ekle ki ödeme başarılı olunca hangi kullanıcının Premium olacağını backend'de yakalayabilelim.3. Backend Serverless Route Oluşturulması (api/paddle-webhook.ts):api/ klasörü altında paddle-webhook.ts adında temiz bir Next.js API route'u oluştur.Bu endpoint'e Paddle'dan gelecek POST isteklerini (Webhook bildirimlerini) dinleyecek yapıyı kur.req.body.event_type verisini kontrol et. Eğer gelen etkinlik 'subscription.created' veya 'transaction.completed' ise, req.body.data.custom_data.userId değerini güvenle oku.Kullanıcı ID'si başarıyla yakalandığında, sistemdeki veritabanı modeline göre kullanıcının isPremium statüsünü true yapacak yorum satırını/kod bloğunu ekle. İsteği başarıyla aldığını belirtmek için Paddle'a HTTP 200 kodu dön.4. Güvenlik ve TypeScript Kontrolleri:Kodları yazarken hiçbir TypeScript tip hatası (any haricinde gerekli yerlerde tip tanımlamaları) kalmadığından emin ol.Değişiklikleri yaparken mevcut UI tasarımını, Tailwind CSS sınıflarını ve Securify'ın genel jilet gibi duran tasarım dilini kesinlikle bozma.
